Tool Use 核心概念

讓 Claude 能執行動作

Tool Use（也叫 Function Calling）是讓 Claude 在回答時，能夠決定「我現在需要呼叫某個函式來取得資訊」。這不是 Claude 自己執行程式碼——Claude 只是告訴你「請幫我執行這個函式，帶這些參數」，你的程式碼負責實際執行，然後把結果回傳給 Claude，Claude 再生成最終回覆。

工具定義
你在呼叫 Claude API 時傳入 tools 陣列，每個工具有 name、description、input_schema。description 要寫清楚，Claude 靠它判斷何時呼叫這個工具。
工具呼叫
Claude 決定要使用工具時，回傳的 stop_reason 是 tool_use，content 裡有 tool_use block，包含 tool name 和 input 參數。
工具結果
你執行工具後，建立一個 role: 'user' 的訊息，content 是 tool_result block，包含 tool_use_id 和執行結果，再送回 Claude。
多工具並行
Claude 可以在單次回應中請求多個工具呼叫（parallel tool use）。你可以用 Promise.all() 並行執行，然後一次性回傳所有結果，減少來回次數。

Tool Use vs. 直接叫 Claude 輸出 JSON

直接輸出 JSON（不用 Tool Use）

適合：Claude 本身有足夠知識可以回答，只是需要結構化輸出。例如分析文字情感、生成行程草稿、提取實體。

缺點：Claude 的知識有截止日期，無法取得即時資料（當前票價、即時天氣）。

Tool Use（呼叫外部 API）

適合：需要即時資料、需要執行動作（搜尋、預訂）、需要整合多個外部服務。

優點：Claude 負責決策和語言生成，外部 API 負責真實數據。分工清晰，可靠性高。

執行迴圈實作

Tool Use 執行迴圈

Tool Use 不是單次來回，而是一個迴圈：Claude 說「我要用工具」→ 你執行工具 → 你把結果給 Claude → Claude 說「我還需要另一個工具」或「我可以回答了」。這個迴圈一直到 Claude 的 stop_reason 是 end_turn 為止。

TOOL USE EXECUTION LOOP
1
APP
呼叫 Claude API，帶入 messages 和 tools 陣列，傳送用戶請求
2
CLAUDE
分析請求，決定需要哪些工具。回傳 stop_reason: "tool_use"，content 包含一或多個 tool_use block（name + input）
3
APP
讀取所有 tool_use blocks，並行呼叫對應的真實 API（Amadeus、Google Maps 等），收集結果
4
EXT API
Amadeus / Google Maps API 回傳真實資料：航班清單、地點列表、路線資訊
5
APP
把 Claude 的 assistant 訊息 + tool_result 訊息（每個工具一筆）加回 messages，再次呼叫 Claude API
6
CLAUDE
讀取工具結果，若需要更多資料重複 2-5，否則生成最終回覆，stop_reason: "end_turn"

完整執行迴圈程式碼

TYPESCRIPT src/lib/tool-loop.ts 
import Anthropic from '@anthropic-ai/sdk'
import { executeTool } from '@/lib/tools/executor'
import { TRAVEL_TOOLS } from '@/lib/tools/travel-tools'
import type { Message } from '@/types'

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY! })

// 回調類型：讓前端可以即時顯示工具執行狀態
type OnToolCall = (name: string, input: unknown) => void
type OnToolResult = (name: string, result: unknown) => void

export async function runTravelAgent(
  userMessages: Message[],
  system: string,
  callbacks?: { onToolCall?: OnToolCall; onToolResult?: OnToolResult }
): Promise<string> {
  // 工作用 messages 陣列，從呼叫者傳入的訊息開始
  const messages: Anthropic.Messages.MessageParam[] = userMessages.map(m => ({
    role: m.role,
    content: m.content,
  }))

  const MAX_ITERATIONS = 10 // 防止無限迴圈
  let iteration = 0

  while (iteration < MAX_ITERATIONS) {
    iteration++

    const response = await anthropic.messages.create({
      model: 'claude-sonnet-4-20250514',
      max_tokens: 4096,
      system,
      tools: TRAVEL_TOOLS,
      messages,
    })

    // 情況 1：Claude 完成，直接回傳文字
    if (response.stop_reason === 'end_turn') {
      const textBlock = response.content.find(b => b.type === 'text')
      return textBlock?.type === 'text' ? textBlock.text : ''
    }

    // 情況 2：Claude 要使用工具
    if (response.stop_reason === 'tool_use') {
      // 把 Claude 這輪的回覆加進 messages
      messages.push({ role: 'assistant', content: response.content })

      // 取出所有 tool_use blocks
      const toolUseBlocks = response.content.filter(
        (b): b is Anthropic.Messages.ToolUseBlock => b.type === 'tool_use'
      )

      // 並行執行所有工具呼叫
      const toolResults = await Promise.all(
        toolUseBlocks.map(async (block) => {
          callbacks?.onToolCall?.(block.name, block.input)
          const result = await executeTool(block.name, block.input)
          callbacks?.onToolResult?.(block.name, result)
          return {
            type: 'tool_result' as const,
            tool_use_id: block.id,
            content: JSON.stringify(result),
          }
        })
      )

      // 把工具結果加進 messages，繼續迴圈
      messages.push({ role: 'user', content: toolResults })
    }
  }

  throw new Error('Tool loop exceeded maximum iterations')
}

工具執行器：路由到真實 API

TYPESCRIPT src/lib/tools/executor.ts 
import { searchFlights } from './amadeus'
import { searchPlaces, getDirections } from './maps'
import { estimateBudget } from './budget'

export async function executeTool(name: string, input: unknown): Promise<unknown> {
  switch (name) {
    case 'search_flights':
      return searchFlights(input as SearchFlightsInput)
    case 'search_places':
      return searchPlaces(input as SearchPlacesInput)
    case 'get_directions':
      return getDirections(input as GetDirectionsInput)
    case 'estimate_budget':
      return estimateBudget(input as EstimateBudgetInput)
    default:
      throw new Error(`Unknown tool: ${name}`)
  }
}
Google Maps 整合

Google Maps API 串接

我們用到兩個 Google Maps API：Places API（搜尋景點餐廳）和 Directions API（路線規劃）。兩者都在伺服器端呼叫，API key 不會暴露到前端。

Google Cloud Console 啟用：Maps JavaScript API、Places API (New)、Directions API、Geocoding API。建立一組 API key，設定 HTTP referrer 限制（加入你的 Vercel domain）。環境變數：GOOGLE_MAPS_API_KEY。

TYPESCRIPT src/lib/tools/maps.ts 
const MAPS_KEY = process.env.GOOGLE_MAPS_API_KEY!

export interface SearchPlacesInput {
  query: string
  location: string
  type?: string
  radius_km?: number
}

export async function searchPlaces(input: SearchPlacesInput) {
  // 先用 Geocoding API 把地名轉座標
  const geoRes = await fetch(
    `https://maps.googleapis.com/maps/api/geocode/json?address=${encodeURIComponent(input.location)}&key=${MAPS_KEY}`
  )
  const geoData = await geoRes.json()
  const { lat, lng } = geoData.results[0].geometry.location

  // 用 Places API 搜尋周邊
  const radius = (input.radius_km ?? 5) * 1000
  const params = new URLSearchParams({
    location: `${lat},${lng}`,
    radius: String(radius),
    keyword: input.query,
    key: MAPS_KEY,
    language: 'zh-TW',
    ...(input.type ? { type: input.type } : {}),
  })

  const res = await fetch(
    `https://maps.googleapis.com/maps/api/place/nearbysearch/json?${params}`
  )
  const data = await res.json()

  // 回傳精簡版資料（避免把大量 JSON 塞給 Claude）
  return data.results.slice(0, 5).map((p: any) => ({
    name: p.name,
    rating: p.rating,
    address: p.vicinity,
    types: p.types.slice(0, 3),
    place_id: p.place_id,
    lat: p.geometry.location.lat,
    lng: p.geometry.location.lng,
  }))
}

export interface GetDirectionsInput {
  origin: string
  destination: string
  mode?: 'transit' | 'walking' | 'driving'
}

export async function getDirections(input: GetDirectionsInput) {
  const params = new URLSearchParams({
    origin: input.origin,
    destination: input.destination,
    mode: input.mode ?? 'transit',
    key: MAPS_KEY,
    language: 'zh-TW',
  })

  const res = await fetch(
    `https://maps.googleapis.com/maps/api/directions/json?${params}`
  )
  const data = await res.json()
  const route = data.routes[0]?.legs[0]

  if (!route) return { error: '找不到路線' }

  return {
    duration: route.duration.text,
    distance: route.distance.text,
    steps: route.steps.slice(0, 5).map((s: any) => ({
      instruction: s.html_instructions.replace(/<[^>]*>/g, ''),
      duration: s.duration.text,
    })),
  }
}

前端 Google Maps 嵌入

用 @react-google-maps/api 套件在前端顯示地圖，把 Claude 取得的景點座標轉成地圖標記，讓用戶直觀看到行程地點分布。

TYPESCRIPT src/components/travel/TripMap.tsx 
'use client'
import { GoogleMap, Marker, useJsApiLoader } from '@react-google-maps/api'

interface Place { name: string; lat: number; lng: number }

export function TripMap({ places }: { places: Place[] }) {
  const { isLoaded } = useJsApiLoader({
    googleMapsApiKey: process.env.NEXT_PUBLIC_GOOGLE_MAPS_KEY!,
  })

  if (!isLoaded || places.length === 0) return null

  const center = { lat: places[0].lat, lng: places[0].lng }

  return (
    <GoogleMap
      mapContainerStyle={{ width: '100%', height: '360px', borderRadius: '4px' }}
      center={center}
      zoom={13}
      options={{ disableDefaultUI: true, zoomControl: true }}
    >
      {places.map((p, i) => (
        <Marker key={i} position={{ lat: p.lat, lng: p.lng }} title={p.name} />
      ))}
    </GoogleMap>
  )
}
Amadeus 航班 API

Amadeus Flight Search 串接

Amadeus 提供真實的全球航班資料，免費帳號可以呼叫 test environment（有真實航班資料，不可實際訂票）。正式上線換到 production API key 即可串接真實預訂。

Amadeus API 需要先取得 access token（OAuth 2.0 Client Credentials flow），token 有效期 1799 秒。需要在伺服器端快取 token，不要每次請求都重新取得。

TYPESCRIPT src/lib/tools/amadeus.ts 
const AMADEUS_BASE = 'https://test.api.amadeus.com'

// 簡易記憶體快取（production 請用 Redis 或 Supabase）
let tokenCache: { token: string; expiresAt: number } | null = null

async function getAccessToken() {
  if (tokenCache && Date.now() < tokenCache.expiresAt - 60000) {
    return tokenCache.token
  }
  const res = await fetch(`${AMADEUS_BASE}/v1/security/oauth2/token`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      grant_type: 'client_credentials',
      client_id: process.env.AMADEUS_API_KEY!,
      client_secret: process.env.AMADEUS_API_SECRET!,
    }),
  })
  const data = await res.json()
  tokenCache = { token: data.access_token, expiresAt: Date.now() + data.expires_in * 1000 }
  return tokenCache.token
}

export interface SearchFlightsInput {
  origin: string
  destination: string
  departure_date: string
  return_date?: string
  adults?: number
}

export async function searchFlights(input: SearchFlightsInput) {
  const token = await getAccessToken()

  const params = new URLSearchParams({
    originLocationCode: input.origin,
    destinationLocationCode: input.destination,
    departureDate: input.departure_date,
    adults: String(input.adults ?? 1),
    max: '5',
    currencyCode: 'TWD',
    ...(input.return_date ? { returnDate: input.return_date } : {}),
  })

  const res = await fetch(
    `${AMADEUS_BASE}/v2/shopping/flight-offers?${params}`,
    { headers: { Authorization: `Bearer ${token}` } }
  )
  const data = await res.json()

  if (!data.data) return { error: '查無航班，請確認機場代碼和日期' }

  // 精簡回傳：避免 Amadeus 原始回應的巨大 JSON 造成 token 浪費
  return data.data.slice(0, 5).map((offer: any) => {
    const seg = offer.itineraries[0].segments[0]
    return {
      airline: seg.carrierCode,
      flight: `${seg.carrierCode} ${seg.number}`,
      departure: seg.departure.at,
      arrival: seg.arrival.at,
      duration: offer.itineraries[0].duration,
      stops: offer.itineraries[0].segments.length - 1,
      price_twd: Math.round(Number(offer.price.total)),
    }
  })
}

Amadeus test API 的 IATA 代碼查詢：台北桃園 = TPE、大阪關西 = KIX、東京成田 = NRT、東京羽田 = HND、巴黎戴高樂 = CDG。在 prompt 中教 Claude 這些常用代碼，減少它猜錯的機率。

React Query 狀態管理

用 React Query 管理工具執行狀態

Tool Use 有一個 UX 問題：用戶送出訊息後，Claude 在後端跑工具迴圈可能需要 5-15 秒。這段時間前端要顯示「正在搜尋航班...」「正在查詢景點...」的進度狀態。我們用 Server-Sent Events（SSE）搭配 React Query 來解決這個問題。

TYPESCRIPT src/app/api/travel/route.ts 
import { NextRequest } from 'next/server'
import { z } from 'zod'
import { runTravelAgent } from '@/lib/tool-loop'
import { TRAVEL_PLANNER_SYSTEM } from '@/lib/prompts/travel'

const TravelSchema = z.object({
  messages: z.array(z.object({ role: z.enum(['user', 'assistant']), content: z.string() })),
})

export async function POST(req: NextRequest) {
  const parsed = TravelSchema.safeParse(await req.json())
  if (!parsed.success) return Response.json({ error: 'Invalid input' }, { status: 400 })

  const encoder = new TextEncoder()

  const stream = new ReadableStream({
    async start(controller) {
      const send = (event: string, data: unknown) =>
        controller.enqueue(encoder.encode(`event: ${event}\ndata: ${JSON.stringify(data)}\n\n`))

      try {
        const result = await runTravelAgent(
          parsed.data.messages,
          TRAVEL_PLANNER_SYSTEM,
          {
            onToolCall: (name, input) => send('tool_call', { name, input }),
            onToolResult: (name, result) => send('tool_result', { name, result }),
          }
        )
        send('done', { text: result })
      } catch (err) {
        send('error', { message: err instanceof Error ? err.message : 'Unknown error' })
      } finally {
        controller.close()
      }
    },
  })

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
    },
  })
}

前端 useTravel Hook

TYPESCRIPT src/hooks/useTravel.ts 
'use client'
import { useState, useCallback } from 'react'
import type { Message } from '@/types'

interface ToolStatus {
  name: string
  status: 'running' | 'done'
  result?: unknown
}

export function useTravel() {
  const [messages, setMessages] = useState<Message[]>([])
  const [toolStatuses, setToolStatuses] = useState<ToolStatus[]>([])
  const [loading, setLoading] = useState(false)
  const [places, setPlaces] = useState<{name:string;lat:number;lng:number}[]>([])

  const ask = useCallback(async (content: string) => {
    const userMsg: Message = { role: 'user', content }
    const nextMessages = [...messages, userMsg]
    setMessages(nextMessages)
    setLoading(true)
    setToolStatuses([])

    const res = await fetch('/api/travel', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ messages: nextMessages }),
    })

    const reader = res.body!.getReader()
    const decoder = new TextDecoder()

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      const lines = decoder.decode(value).split('\n')
      let currentEvent = ''
      for (const line of lines) {
        if (line.startsWith('event: ')) currentEvent = line.slice(7)
        if (!line.startsWith('data: ')) continue
        const payload = JSON.parse(line.slice(6))

        if (currentEvent === 'tool_call') {
          setToolStatuses(prev => [...prev, { name: payload.name, status: 'running' }])
        }
        if (currentEvent === 'tool_result') {
          setToolStatuses(prev => prev.map(t =>
            t.name === payload.name ? { ...t, status: 'done', result: payload.result } : t
          ))
          // 若是地點結果，更新地圖
          if (payload.name === 'search_places' && Array.isArray(payload.result)) {
            setPlaces(payload.result)
          }
        }
        if (currentEvent === 'done') {
          setMessages(prev => [...prev, { role: 'assistant', content: payload.text }])
          setLoading(false)
        }
      }
    }
  }, [messages])

  return { messages, toolStatuses, loading, places, ask }
}

旅遊行程資料庫 Schema

在 Class 01 的 conversations/messages 基礎上，加入旅遊行程專用的表格：trip（行程主體）和 trip_places（行程中的地點）。這讓用戶可以儲存、重新編輯、分享行程。

SQL supabase/migrations/002_travel.sql 
-- ===== TRIPS =====
create table trips (
  id            uuid default gen_random_uuid() primary key,
  user_id       uuid references profiles(id) on delete cascade not null,
  title         text not null,
  destination   text not null,
  start_date    date,
  end_date      date,
  budget_twd    integer,
  travelers     integer default 1,
  style         text check(style in('budget', 'mid-range', 'luxury')),
  summary       text,                         -- Claude 生成的行程摘要
  raw_itinerary jsonb,                         -- 完整行程 JSON
  created_at    timestamptz default now() not null
);

-- ===== TRIP_PLACES =====
create table trip_places (
  id          uuid default gen_random_uuid() primary key,
  trip_id     uuid references trips(id) on delete cascade not null,
  name        text not null,
  place_id    text,                            -- Google Maps place_id
  lat         numeric(10, 7),
  lng         numeric(10, 7),
  visit_date  date,
  visit_order integer,
  duration_min integer,
  notes       text,
  created_at  timestamptz default now() not null
);

create index idx_trips_user_id on trips(user_id);
create index idx_trip_places_trip_id on trip_places(trip_id);

-- ===== RLS =====
alter table trips       enable row level security;
alter table trip_places enable row level security;

create policy "trips: own" on trips
  for all using (auth.uid() = user_id);

create policy "trip_places: own trips" on trip_places
  for all using (
    trip_id in (select id from trips where user_id = auth.uid())
  );

-- 儲存行程的函式（在 Claude 生成完畢後呼叫）
create or replace function save_trip_from_ai(
  p_user_id     uuid,
  p_title       text,
  p_destination text,
  p_summary     text,
  p_itinerary   jsonb,
  p_places      jsonb   -- [{name, lat, lng, visit_date, visit_order}]
) returns uuid as $$
declare
  v_trip_id uuid;
begin
  insert into trips(user_id, title, destination, summary, raw_itinerary)
  values(p_user_id, p_title, p_destination, p_summary, p_itinerary)
  returning id into v_trip_id;

  insert into trip_places(trip_id, name, lat, lng, visit_date, visit_order)
  select v_trip_id, p->>'name', (p->>'lat')::numeric, (p->>'lng')::numeric,
         (p->>'visit_date')::date, (p->>'visit_order')::integer
  from jsonb_array_elements(p_places) as p;

  return v_trip_id;
end;
$$ language plpgsql security definer;
