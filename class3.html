智慧餐飲
點餐系統

這堂課要解決一個真實問題：餐廳點餐的資訊流動——從顧客點餐、AI 推薦搭配、廚房即時接單、到結帳付款，每個環節都要即時同步。核心技術是 Supabase Realtime，它讓你不需要自己架設 WebSocket，資料庫的每一次寫入都能即時推送到所有連線的客戶端。

Supabase Realtime Web Speech API Stripe Claude API Next.js 15 TypeScript PostgreSQL
LEARNING OUTCOME
課程結束後，你擁有：(1) 一個完整的餐飲點餐系統，顧客端、廚房顯示板、管理後台三端同步，訂單狀態更新在 100ms 內反映到所有畫面；(2) 語音點餐功能，說出「一份牛排七分熟，一杯紅酒，不要麵包」，AI 解析並自動加入購物車；(3) Stripe Checkout 整合，可以用真實信用卡付款（test mode）。
系統架構

三端同步架構

這個系統有三個使用者角色，各有獨立的介面，但共用同一個 Supabase 資料庫。資料流向是單向的：顧客下單 → 廚房接單執行 → 管理端監控統計。Realtime 讓三端的狀態永遠保持一致。

CUSTOMER
顧客端：瀏覽菜單、語音/文字點餐、AI 推薦搭配、加入購物車、Stripe 付款、即時追蹤訂單狀態
KITCHEN
廚房顯示板（KDS）：即時顯示新訂單、標記每道菜的製作狀態（待製作→製作中→完成）、無需刷新頁面
ADMIN
管理後台：即時訂單監控、庫存管理、菜單上下架、今日營收統計、熱門菜品分析
Supabase Realtime
基於 PostgreSQL 的 LISTEN/NOTIFY 機制，透過 WebSocket 推送資料庫變更到前端。不需要自己維護 WebSocket server，Supabase 幫你做好所有基礎設施。
Web Speech API
瀏覽器原生語音辨識 API，不需要第三方服務。辨識結果是文字，再交給 Claude 解析成結構化的點餐資料（菜名、數量、備注）。
Stripe Checkout
最簡單的 Stripe 整合方式：後端建立 Checkout Session，前端 redirect 到 Stripe 托管的付款頁面。Stripe 處理所有 PCI DSS 合規問題，你不需要碰信用卡號碼。
Optimistic Updates
加入購物車、標記完成等操作，先更新 UI 讓使用者感覺即時，再在背景寫入資料庫。失敗時 rollback。這讓系統感覺比實際速度快 200-400ms。
AI 菜單推薦

根據情境推薦菜單搭配

AI 推薦不是「隨機顯示熱門菜品」。它要根據用戶已點的菜、當前時段、庫存狀態、以及菜單上的口味描述，給出有邏輯的搭配建議。Claude 的語言理解能力在這裡是核心優勢。

TYPESCRIPT src/app/api/menu/recommend/route.ts 
import { NextRequest, NextResponse } from 'next/server'
import { z } from 'zod'
import Anthropic from '@anthropic-ai/sdk'
import { createClient } from '@/lib/supabase/server'

const Schema = z.object({
  cart_items: z.array(z.object({
    name: z.string(),
    category: z.string(),
  })),
  table_size: z.number().optional(),
})

export async function POST(req: NextRequest) {
  const parsed = Schema.safeParse(await req.json())
  if (!parsed.success) return NextResponse.json({ error: 'Invalid input' }, { status: 400 })

  const supabase = await createClient()

  // 取得有庫存的菜單（stock > 0）
  const { data: menuItems } = await supabase
    .from('menu_items')
    .select('name, category, description, price, tags')
    .eq('available', true)
    .gt('stock', 0)

  const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY! })

  const cartDesc = parsed.data.cart_items.length > 0
    ? `顧客已點：${parsed.data.cart_items.map(i => i.name).join('、')}`
    : '顧客尚未點任何菜'

  const res = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 512,
    system: `你是一個高級餐廳的服務生 AI。根據顧客已點的菜和完整菜單，推薦 2-3 道最適合搭配的菜品或飲料。
回覆 JSON 格式：{"recommendations": [{"name": "菜名", "reason": "30字內的推薦理由"}]}`,
    messages: [{
      role: 'user',
      content: `${cartDesc}\n\n完整菜單：\n${JSON.stringify(menuItems, null, 2)}`,
    }],
  })

  const text = res.content[0].type === 'text' ? res.content[0].text : '{}'

  try {
    const result = JSON.parse(text)
    return NextResponse.json(result)
  } catch {
    return NextResponse.json({ recommendations: [] })
  }
}

在 system prompt 要求 Claude 只推薦菜單上存在的菜名（把完整菜單傳入），否則 Claude 會「創造」不存在的菜品。這是 AI 推薦系統最常見的問題之一。

語音點餐

Web Speech API + Claude 意圖解析

Web Speech API 做語音辨識，Claude 做意圖解析。語音辨識只能給你一段文字，Claude 把這段文字轉成結構化的點餐資料：菜名（對應到菜單）、數量、備注（少辣、不加蔥等）。

Web Speech API 目前在 Chrome/Edge 支援最好，Safari 部分支援，Firefox 需要實驗性旗標。商業部署建議加 fallback：偵測不支援時顯示一般文字輸入。

'use client'
import { useState, useRef, useCallback } from 'react'

declare global {
  interface Window { SpeechRecognition: unknown; webkitSpeechRecognition: unknown }
}

export interface ParsedOrder {
  items: { name: string; qty: number; notes?: string }[]
  raw_text: string
}

export function useVoiceOrder(menuNames: string[]) {
  const [listening, setListening] = useState(false)
  const [transcript, setTranscript] = useState('')
  const [parsing, setParsing] = useState(false)
  const recRef = useRef<unknown>(null)

  const startListening = useCallback(() => {
    const SpeechRecognition =
      window.SpeechRecognition || window.webkitSpeechRecognition
    if (!SpeechRecognition) {
      alert('你的瀏覽器不支援語音辨識，請使用 Chrome')
      return
    }

    const rec = new (SpeechRecognition as any)()
    rec.lang = 'zh-TW'
    rec.continuous = false
    rec.interimResults = false
    recRef.current = rec

    rec.onstart = () => setListening(true)
    rec.onend = () => setListening(false)
    rec.onresult = (e: any) => {
      const text = e.results[0][0].transcript
      setTranscript(text)
    }
    rec.start()
  }, [])

  const stopListening = useCallback(() => {
    (recRef.current as any)?.stop()
  }, [])

  // 用 Claude 解析語音文字成結構化點餐
  const parseOrder = useCallback(async (text: string): Promise<ParsedOrder> => {
    setParsing(true)
    try {
      const res = await fetch('/api/menu/parse-voice', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ text, menu_names: menuNames }),
      })
      return res.json()
    } finally {
      setParsing(false)
    }
  }, [menuNames])

  return { listening, transcript, parsing, startListening, stopListening, parseOrder }
}

Supabase Realtime

即時訂單狀態同步

Supabase Realtime 有三種訂閱模式：Broadcast（手動推送訊息）、Presence（追蹤線上用戶）、Postgres Changes（監聽資料庫變更）。我們主要用 Postgres Changes：只要 orders 或 order_items 表格有任何 INSERT/UPDATE，所有訂閱的客戶端都會即時收到通知。

REALTIME ORDER FLOW
1
顧客端
顧客確認購物車 → 呼叫 /api/orders → 寫入 orders + order_items 表格
2
Supabase
PostgreSQL WAL（Write-Ahead Log）捕捉到 INSERT 事件，透過 Realtime server 推送到所有訂閱者
3
廚房端
即時收到新訂單，自動出現在 KDS 畫面最上方，附帶聲音提示，無需廚師手動刷新
4
廚房端
廚師標記每道菜「製作中」→「完成」，UPDATE order_items.status
5
顧客端
即時看到訂單進度更新（等待中 → 製作中 → 送餐中），不需要 polling
TYPESCRIPT src/hooks/useOrderRealtime.ts 
'use client'
import { useEffect, useState } from 'react'
import { createBrowserClient } from '@supabase/ssr'
import type { Order } from '@/types'

export function useOrderRealtime(orderId: string) {
  const [order, setOrder] = useState<Order | null>(null)

  useEffect(() => {
    const supabase = createBrowserClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
    )

    // 先取得目前訂單狀態
    supabase
      .from('orders')
      .select('*, order_items(*, menu_items(name, image_url))')
      .eq('id', orderId)
      .single()
      .then(({ data }) => data && setOrder(data as Order))

    // 訂閱 order_items 變更（廚房更新製作狀態時觸發）
    const channel = supabase
      .channel(`order-${orderId}`)
      .on(
        'postgres_changes',
        {
          event: 'UPDATE',
          schema: 'public',
          table: 'order_items',
          filter: `order_id=eq.${orderId}`,
        },
        async () => {
          // 重新取得完整訂單（包含所有 items 的最新狀態）
          const { data } = await supabase
            .from('orders')
            .select('*, order_items(*, menu_items(name, image_url))')
            .eq('id', orderId)
            .single()
          if (data) setOrder(data as Order)
        }
      )
      .subscribe()

    return () => { supabase.removeChannel(channel) }
  }, [orderId])

  return order
}

廚房端：監聽所有新訂單

TYPESCRIPT src/hooks/useKitchenOrders.ts 
'use client'
import { useEffect, useState, useCallback } from 'react'
import { createBrowserClient } from '@supabase/ssr'
import type { Order } from '@/types'

export function useKitchenOrders() {
  const [orders, setOrders] = useState<Order[]>([])
  const supabase = createBrowserClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )

  useEffect(() => {
    // 取得今日所有未完成訂單
    supabase
      .from('orders')
      .select('*, order_items(*, menu_items(name))')
      .in('status', ['paid', 'preparing'])
      .gte('created_at', new Date().toISOString().split('T')[0])
      .order('created_at', { ascending: true })
      .then(({ data }) => data && setOrders(data as Order[]));

    // 監聽新訂單進來（INSERT）
    const channel = supabase
      .channel('kitchen-orders')
      .on(
        'postgres_changes',
        { event: 'INSERT', schema: 'public', table: 'orders' },
        async (payload) => {
          // 取得新訂單的完整資訊（含 order_items）
          const { data } = await supabase
            .from('orders')
            .select('*, order_items(*, menu_items(name))')
            .eq('id', payload.new.id)
            .single()
          if (data) {
            setOrders(prev => [data as Order, ...prev])
            // 播放提示音
            new Audio('/sounds/new-order.mp3').play().catch(() => {})
          }
        }
      )
      .on(
        'postgres_changes',
        { event: 'UPDATE', schema: 'public', table: 'orders' },
        (payload) => {
          setOrders(prev =>
            payload.new.status === 'completed'
              ? prev.filter(o => o.id !== payload.new.id)   // 完成的訂單移出廚房看板
              : prev.map(o => o.id === payload.new.id ? { ...o, ...payload.new } : o)
          )
        }
      )
      .subscribe()

    return () => { supabase.removeChannel(channel) }
  }, [])

  const markItemDone = useCallback(async (itemId: string) => {
    await supabase
      .from('order_items')
      .update({ status: 'done', done_at: new Date().toISOString() })
      .eq('id', itemId)
  }, [supabase])

  return { orders, markItemDone }
}

Stripe 結帳

Stripe Checkout 整合

Stripe Checkout 是最快的付款整合方式：你在後端建立 Checkout Session，前端 redirect 到 Stripe 的托管付款頁面，付款完成後 Stripe redirect 回你的網站。整個流程不需要你自己處理信用卡資料。

Stripe test mode：使用 4242 4242 4242 4242（任意有效期 + 任意 CVV）可以模擬成功付款。Dashboard 可以看到所有 test 交易記錄，方便開發期間測試。

TYPESCRIPT src/app/api/checkout/route.ts 
import { NextRequest, NextResponse } from 'next/server'
import Stripe from 'stripe'
import { z } from 'zod'
import { createClient } from '@/lib/supabase/server'

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!)

const Schema = z.object({
  order_id: z.string().uuid(),
  table_number: z.number(),
})

export async function POST(req: NextRequest) {
  const parsed = Schema.safeParse(await req.json())
  if (!parsed.success) return NextResponse.json({ error: 'Invalid input' }, { status: 400 })

  const supabase = await createClient()

  // 取得訂單明細（用來建立 Stripe line_items）
  const { data: order } = await supabase
    .from('orders')
    .select('*, order_items(quantity, unit_price, notes, menu_items(name))')
    .eq('id', parsed.data.order_id)
    .single()

  if (!order) return NextResponse.json({ error: 'Order not found' }, { status: 404 })

  const lineItems = order.order_items.map((item: any) => ({
    price_data: {
      currency: 'twd',
      unit_amount: Math.round(item.unit_price),  // TWD 以最小單位（元）計算
      product_data: {
        name: item.menu_items.name,
        ...(item.notes ? { description: item.notes } : {}),
      },
    },
    quantity: item.quantity,
  }))

  const baseUrl = process.env.NEXT_PUBLIC_BASE_URL
  const session = await stripe.checkout.sessions.create({
    mode: 'payment',
    line_items: lineItems,
    metadata: {
      order_id: parsed.data.order_id,
      table_number: String(parsed.data.table_number),
    },
    success_url: `${baseUrl}/order/${parsed.data.order_id}?paid=1`,
    cancel_url: `${baseUrl}/order/${parsed.data.order_id}`,
  })

  return NextResponse.json({ url: session.url })
}

Stripe Webhook — 付款成功後更新訂單狀態

TYPESCRIPT src/app/api/webhooks/stripe/route.ts 
import { NextRequest, NextResponse } from 'next/server'
import Stripe from 'stripe'
import { createClient } from '@/lib/supabase/server'

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!)

export async function POST(req: NextRequest) {
  const body = await req.text()
  const sig = req.headers.get('stripe-signature')!

  let event: Stripe.Event
  try {
    event = stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)
  } catch {
    return NextResponse.json({ error: 'Invalid signature' }, { status: 400 })
  }

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object as Stripe.Checkout.Session
    const orderId = session.metadata?.order_id
    if (!orderId) return NextResponse.json({ ok: true })

    // 更新訂單狀態為 paid → 廚房 Realtime 即時收到通知
    const supabase = await createClient()
    await supabase
      .from('orders')
      .update({
        status: 'paid',
        paid_at: new Date().toISOString(),
        stripe_session_id: session.id,
      })
      .eq('id', orderId)
  }

  return NextResponse.json({ ok: true })
}

本地開發測試 Webhook：用 Stripe CLI 指令 stripe listen --forward-to localhost:3000/api/webhooks/stripe，它會把 Stripe 的 webhook 事件轉發到本地端。取得本地 webhook secret 設為 STRIPE_WEBHOOK_SECRET。

-- ===== MENU_ITEMS =====
create table menu_items (
  id          uuid default gen_random_uuid() primary key,
  name        text not null,
  description text,
  price       numeric(10, 0) not null,             -- 台幣整數
  category    text not null,                        -- 前菜、主菜、甜點、飲料
  image_url   text,
  tags        text[] default '{}',                  -- 素食、海鮮、辣、招牌...
  available   boolean default true not null,
  stock       integer default 99,                   -- -1 = 無限量
  created_at  timestamptz default now()
);

-- ===== ORDERS =====
create table orders (
  id               uuid default gen_random_uuid() primary key,
  user_id          uuid references profiles(id) on delete set null,  -- 允許匿名點餐
  table_number     integer,
  status           text not null default 'pending'  -- pending/paid/preparing/completed/cancelled
                   check(status in('pending','paid','preparing','completed','cancelled')),
  total_amount     numeric(10, 0) not null,
  stripe_session_id text,
  notes            text,
  paid_at          timestamptz,
  created_at       timestamptz default now() not null
);

-- ===== ORDER_ITEMS =====
create table order_items (
  id             uuid default gen_random_uuid() primary key,
  order_id       uuid references orders(id) on delete cascade not null,
  menu_item_id   uuid references menu_items(id) not null,
  quantity       integer not null check(quantity > 0),
  unit_price     numeric(10, 0) not null,             -- 快照當時價格，避免後來改價影響舊訂單
  notes          text,                                 -- 備注：少辣、不加蔥...
  status         text not null default 'pending'
                 check(status in('pending', 'preparing', 'done')),
  done_at        timestamptz,
  created_at     timestamptz default now() not null
);

create index idx_orders_status     on orders(status);
create index idx_orders_created_at on orders(created_at);
create index idx_order_items_order on order_items(order_id);

-- 或用 SQL：
alter publication supabase_realtime add table orders;
alter publication supabase_realtime add table order_items;

-- ===== RLS =====
alter table menu_items  enable row level security;
alter table orders      enable row level security;
alter table order_items enable row level security;

-- 菜單：所有人可讀，只有 admin 可寫
create policy "menu: public read" on menu_items for select using (true);
create policy "menu: admin write" on menu_items for all using
  (auth.uid() in (select id from profiles where role = 'admin'));

-- 訂單：自己的訂單可讀，廚房 service role 可讀所有並更新
create policy "orders: own read" on orders for select using
  (user_id = auth.uid() or auth.uid() is null);
create policy "orders: insert own" on orders for insert with check (true);  -- 允許匿名建立

-- 庫存扣減 trigger
create or replace function deduct_stock() returns trigger as $$
begin
  update menu_items
  set stock = stock - NEW.quantity
  where id = NEW.menu_item_id and stock > 0;
  return NEW;
end; $$ language plpgsql;

create trigger on_order_item_insert
  after insert on order_items
  for each row execute procedure deduct_stock();
